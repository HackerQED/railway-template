# Agent 套壳 SaaS 思路

## 核心想法

任何「Agent + Skill」能完成的事情，都可以包装成 SaaS 来卖。

公式：**SaaS = Agent（Agent 壳）+ Skill（能力定义）+ 后端 API（实际生成）+ 付费包装（网站）**

## 为什么成立

1. **开发成本极低**：不需要自己做 chat UI、上下文管理、工具编排——Agent 全包
2. **迭代成本极低**：改产品 = 改 Skill 文件，不需要改前端代码
3. **双轨分发**：
   - 有 Agent 的用户 → 直接用 Skill（免费引流，只收 API 费）
   - 没有的用户 → 用我们的 SaaS 网站（收订阅费 + API 费）
4. **和 Agent First 方向一脉相承**：不是新方向，是同一条路的自然延伸
5. **核心竞争力是流量能力**，不是技术差异化。做套壳已一年多，利润模型清楚

## 用户分层策略

### 免费用户
- 自己起一个 Agent 公开实例，免费用户共享使用
- 资源是 share 的，用户能接受
- 用户不需要自己安装 Skill，直接在网站界面上就能用，路径最短

### 付费用户（两种路径）
1. **直接给用户部署**独立实例
2. **自己找供应商**，用一台机器部署，不一定要按请求计费

### 长期选项：自己做 Agent 供应商
目前市面上基本都是按机器售卖，没有服务于开发者和套壳客户的按请求模型。这个方向可以做，但不是第一步——先上网找现有供应商，找不到合适的再自己做。

## 状态

idea 阶段，非 MVP 必需。先把 Skill 本身调好，这个思路自然就能落地。
