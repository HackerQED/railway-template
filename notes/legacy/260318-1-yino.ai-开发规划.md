# 260318 - yino.ai 开发规划

## 方向

从内嵌 Agent 的 MV 生成器，转向 **All-in-One AI Video & Image Generator**。

核心思路（详见 `260317 - Agent First 的 All in One 视频图片站.md`）：
- 先做一个**人能用的表单站**，这是我们做过两遍的东西（wan2、onesite），第三遍做得更好
- Agent 是外挂（Skill + API），不是内嵌
- "正确做 Agent 的方法是不做 Agent"

## MVP：以 Music Video 切入

为什么先打 MV：
- 有完整的增长方法论（之前调研过竞品、验证过 Agent 编排）
- 可以同步发布 Music Video Skill，出去宣传，打 "openclaw video" 等 SEO 关键词
- MV 链路天然串起 4 个核心能力：音乐分析 → 图片生成 → 视频生成 → 拼合

### 4 个核心能力

| 能力 | 实现 | 说明 |
|------|------|------|
| 音乐分析 | Gemini multimodal | 分析节拍、情绪、段落 |
| 图片生成 | Seedream 4.5 / 5 | 关键帧 |
| 视频生成 | Veo 3.1 | 图转视频 |
| 视频拼合 | FFmpeg（后端） | 通用拼合接口，不限于 MV |

## URL 设计

三个维度交叉引流：功能页吃用户意图词，模型页吃模型名词，SEO 页吃 Agent 平台词。

```
yino.ai/
│
├── 功能页（按用户意图，每个页面自带生成器表单）
│   ├── /music-video-generator
│   ├── /image-generator
│   ├── /text-to-video
│   ├── /image-to-video
│   └── /video-composer
│
├── 模型页（按模型名，吃模型相关搜索词）
│   ├── /models/seedream
│   ├── /models/seedream-5
│   ├── /models/veo-3
│   ├── /models/veo-31
│   └── /models/...
│
├── Skill 页（按功能分，不按 Agent 分）
│   ├── /skills                          # Skill 目录总览
│   ├── /skills/music-video-generator    # MV 生成 Skill
│   ├── /skills/image-generator          # 图片生成 Skill
│   └── /skills/video-generator          # 视频生成 Skill
│
├── SEO 着陆页（按 Agent 平台 × 功能，打具体关键词）
│   ├── /openclaw-video-skill
│   ├── /openclaw-music-video
│   ├── /openclaw-create-video
│   ├── /claude-code-video
│   └── /codex-video
│
├── 用户页
│   ├── /gallery                         # 作品展示 / 灵感
│   ├── /dashboard                       # 生成历史
│   ├── /settings                        # 账号设置
│   └── /pricing
│
├── 标准页
│   ├── /blog
│   ├── /docs                            # API 文档
│   ├── /privacy
│   └── /terms
│
└── Agent 入口
    ├── /llms.txt                        # Agent 发现入口
    └── /api/capabilities                # API 能力发现
```

### 交叉引流逻辑

- **SEO 着陆页**（如 `/openclaw-video-skill`）→ 列出所有 Skill + 功能页链接
- **Skill 页**（如 `/skills/music-video-generator`）→ 列出可用模型 + 安装方式（不区分 Agent）
- **模型页**（如 `/models/seedream`）→ 提到可以通过 Skill 或表单使用
- **功能页**（如 `/music-video-generator`）→ 内嵌生成器 + 列出可用模型 + 提到 Skill

## 原则

- **表单站优先**：人能直接用，这是基本盘
- **API 是核心**：表单和 Skill 都是同一套 API 的两层皮
- **Agent 供应商中立**：OpenClaw / Claude Code / Codex 都支持，不锁定平台
- **Provider 中立**：不暴露上游提供商信息
- **三维度 SEO**：功能页 × 模型页 × Agent 平台页，交叉引流
- **部署在 Railway**：Docker 环境，可以跑 FFmpeg
