# Preview Block 布局层级设计

> **日期**: 2026-03-21
> **状态**: ~~初步雏形~~ → 已被 260324-1 的两层结构取代（三层简化为两层，去掉了 L3 Tab 切换和 group 嵌套）

## 核心理解

整个 Preview 是一份**结构化文档**，不是散乱的卡片堆砌：

- **L1 = 章节**：文档的一级标题 + 下属内容
- **L2 = 卡片**：内容的最小展示单元，带边框和背景
- **L3 = 卡片内部切换**：Tab 切换，不影响外层布局

## 各层级布局规则

### L1 — 章节层

- 大标题 + 描述文字
- 子内容排列在网格中（`grid-cols-1 sm:grid-cols-2 lg:grid-cols-3`）
- **不直接放媒体**，媒体应归入 Group 后以 L2 卡片形式展现
- 如果确实有独立媒体出现在 L1（兜底场景），使用 constrained 模式限制高度

### L2 — 卡片层

- 圆角边框 + `bg-card` 背景
- 结构为**三段式**（目前只用了前两段）：
  1. **Header**: 标题 + 描述 + 操作按钮（@ mention 等）
  2. **Media**: 固定高度（`h-48`）+ `object-contain`，保证同行卡片等高
  3. **Footer**: （预留）未来放控件——如 Regenerate、Edit、时长、prompt 等
- 媒体区域用 `object-contain` 而非 `object-cover`，保持原始比例不裁切
- `bg-muted` 填充留白区域

### L3 — Tab 切换层

- 当 Group 内有多个 media block 时，自动启用 Tab 切换
- Tab 标签用媒体类型图标（图片/视频/音频）区分
- 不影响外层卡片尺寸，所有 tab 内容共享同一个容器

## 当前已解决的问题

1. **L1 视频撑爆页面**：所有媒体统一使用 constrained 模式（固定高度 + object-contain）
2. **L2 同行卡片高度不一致**：媒体区域固定 `h-48`，不随内容比例变化

## 待思考的问题

1. **L2 卡片 Footer 区域**：参考 BeatViz 的设计，未来可能需要放置操作控件（Regenerate、Edit、时长显示等），目前结构上尚未预留 slot
2. **固定高度值的选择**：当前 `h-48`（192px）是否足够？不同屏幕尺寸是否需要响应式调整？
3. **L1 独立媒体的定位**：理想情况下 L1 不应该直接放媒体，但 Agent 生成的 blocks 结构可能不完全符合预期，需要考虑兜底体验
4. **视频播放体验**：卡片内的视频 controls 在小尺寸下是否好用？是否需要点击放大播放？
5. **音频的展示方式**：音频没有视觉内容，在固定高度容器里显得空旷，可能需要特殊处理
